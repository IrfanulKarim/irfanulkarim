Sharing one of the conflict experiences from Salient, where I worked with 3 teams: the Salesforce team, the Backend team, and the Mobile team. It was often that the SF team and the backend team had conflicts related to API development. The main issue was that the backend team wasn't developing the API according to the SF team's requirements. Here's how I handled it:

  • I conducted 1:1 meetings with each team member.
  • Performed a root cause analysis.
  • Discovered that the root cause was unclear requirements, so I established a simple communication channel to ensure everyone was on the same page, promoting transparency.
  • I asked the SF team to prepare a sample JSON format to clearly express their requirements. This allowed the backend team to develop the API following the desired format.
  • These steps helped reduce confusion and conflicts between both teams.
